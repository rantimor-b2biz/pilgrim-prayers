## 🚀 Quick Start

### **I'm starting a new week of content**
1. Go to: `01-briefs/week-[X]/`
2. Read: `[DATE]-week-[X]-garden-tomb-brief.md` (MAIN BRIEF)
3. Share: Agent briefs with your team

### **I'm an agent delivering work**
1. Create your deliverable in: `02-deliverables/week-[X]-deliverables/[agent-name]/`
2. Use filename: `[DATE]-[topic]-[type].md`
3. Example: `2026-02-24-garden-tomb-post.md`

### **I need to publish content**
1. Get Gatekeeper approval
2. Save final version to: `03-published/blog-posts/2026-02/week-[X]/`
3. Filename: `[topic]-[type].md`
4. Example: `garden-tomb-quiet-morning.md`

### **I'm creating an email campaign**
1. After blog is published
2. Create email version
3. Save to: `04-campaigns/email/2026-02/`

## 🔄 Content Workflow

```
PLAN (01-briefs/)
  ↓ Create brief & timeline
PRODUCE (02-deliverables/)
  ↓ Agents create content
REVIEW (02-deliverables/gatekeeper/)
  ↓ Gatekeeper approves
PUBLISH (03-published/)
  ↓ Blog post goes live
CAMPAIGN (04-campaigns/)
  ↓ Email campaign launches
ARCHIVE
  ↓ Everything stays for reference
```

---

## ✅ Rules & Best Practices

✅ **DO:**
- Save briefs in `01-briefs/` before agents start
- Save agent work in `02-deliverables/` while in progress
- Move final content to `03-published/` after Gatekeeper approval
- Use date-based naming: `YYYY-MM-DD-[subject].md`
- Keep old posts in `05-archive/` for reference

❌ **DON'T:**
- Save files directly in `O-output/` root
- Delete old posts or campaigns
- Overwrite draft versions
- Skip Gatekeeper review
- Mix agent work with published content
