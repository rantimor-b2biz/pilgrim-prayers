# UX Expert ↔ Paid Search Agent Collaboration

## Executive Summary

The **UX Expert Agent** and **Paid Search Agent** work together to convert searchers into prayer submitters. UX owns the landing page experience (what happens AFTER someone clicks an ad). Paid Search owns the ad targeting and messaging (getting them to click). They must stay synchronized to ensure:

1. **Message Alignment** — Ad promises match landing page reality
2. **Data Sharing** — Friction insights inform ad strategy; performance data informs UX priorities
3. **Audience Insights** — Keywords reveal user intent; user behavior reveals keyword performance gaps

---

## Clear Division of Responsibilities

### UX Expert Agent Owns

**Landing Pages & Conversion Funnels:**
- Page design, layout, and visual hierarchy
- Form field order, microcopy, and validation messaging
- Mobile experience (responsive design, touch targets, load speed)
- Trust signals placement (testimonials, proof of placement, security badges)
- Conversion funnel analysis — where visitors drop off and why

**Speed & Performance:**
- Page load time (<3 seconds)
- Image optimization and compression
- CSS/JS minification and caching
- Core Web Vitals monitoring

**A/B Testing (UX Elements):**
- CTA button text, color, and placement
- Hero image variations
- Form field count and progression
- Testimonial placement
- Page layout and content order

**Monthly Deliverables:**
- Friction analysis reports (where users get stuck)
- A/B test results with conversion impact
- Mobile audit with performance scores
- Monthly UX performance summary with recommendations

---

### Paid Search Agent Owns

**Ad Strategy & Keywords:**
- Keyword research and intent mapping (4 tiers: direct, faith-seeking, gift, site-specific)
- Ad group structure and keyword-to-page alignment
- Negative keywords to reduce wasted spend
- Seasonal keyword planning

**Ad Copy & Creative:**
- Search ad headlines (30 chars max)
- Search ad descriptions (90 chars max)
- Ad copy variants and A/B testing
- Display ad creative direction (aligned with Visual Agent)
- Remarketing audience creation and messaging

**Campaign Performance:**
- Traffic volume and source analysis
- Cost per acquisition (CPA) by campaign, keyword, geo
- Click-through rate (CTR) and conversion rate trends
- Quality Score monitoring
- Budget allocation and reallocation

**Weekly Deliverables:**
- Search terms report with keyword expansion opportunities
- Performance analysis (traffic quality, conversion trends)
- Ad copy variants based on content insights
- Weekly performance summary with optimization recommendations

| Area | UX Owns | PS Owns | How They Collaborate |
|------|---------|---------|----------------------|
| **Message Clarity** | Page headline clarity, copy tone | Ad headline, description | PS validates page matches ad promise weekly |
| **Conversion Rate** | Form design, friction reduction | Traffic quality, audience targeting | Both track; UX optimizes funnel, PS optimizes targeting |
| **Mobile Performance** | Mobile page design, speed, UX | Mobile ad strategy, mobile-specific campaigns | Coordinate on mobile-first strategy if mobile converts better |

## Weekly Sync Protocol

### **Monday — UX Briefing (UX → PS)**

**What UX shares:**
- Top 3 friction points from last week (form abandonment, page speed, trust gaps)
- A/B test results from previous week (winner, confidence, impact on conversion)
- Biggest funnel drop-off (which step lost the most visitors)
- Mobile vs desktop performance trends

**Why it matters to Paid Search:**
- If page is slow → traffic quality issue (users bouncing)
- If form abandonment is high → ad messaging may not set expectations correctly
- If mobile converts much worse → need mobile-only campaign or dedicated mobile landing page

**Format:**
```
MONDAY UX BRIEF
Last Week's Friction Points:
1. [Friction point] — Impact: [conversion drop %] — Fix: [in progress]
2. [Friction point]
3. [Friction point]

Last Week's A/B Test:
- Test: [variable tested]
- Winner: [winning variant]
- Confidence: [statistical significance]
- Impact: [+X% conversion]
- Action: [rolling out / investigating further]

Funnel Health:
- Step 1 (Landing page): 100% → Step 2 (Scroll hero): XX%
- Biggest drop: [Step X] — [XX% drop] — Hypothesis: [user insight]

Alerts:
- Mobile conversion down 5% (investigating page speed)
- Form abandonment up 3% (testing reduced fields)
```

---

### **Wednesday — Paid Search Briefing (PS → UX)**

**What Paid Search shares:**
- Traffic volume by campaign and landing page (which keywords drive traffic to which pages)
- Conversion rates by landing page (which pages convert best, which need help)
- Quality Score issues (if landing page experience is dragging down score)
- New keywords to plan landing page variants for
- Audience insights from search terms report (what are people actually searching for?)

**Why it matters to UX:**
- If a page gets lots of traffic but low conversion → design/speed/messaging issue
- If a keyword drives traffic to wrong landing page → mismatch in intent
- If Quality Score is low on landing page experience → page speed, mobile, or content match issue
- If new keywords are emerging → need to ensure landing pages cover those intent types

**Format:**
```
WEDNESDAY PAID SEARCH BRIEF
Traffic & Conversion by Landing Page:
- [Page Name]: [X impressions] → [Y clicks] → [Z conversions] = [Z% conversion]
- [Page Name]: [trending up/down]
- [Page Name]: [QS issue noted]

Quality Score Alerts:
- [Campaign]: Average QS [X] — Issue: [landing page experience / ad relevance / CTR]
- Action: [UX to improve speed] / [PS to refine keywords]

Emerging Keywords:
- "[New keyword]" — [X searches/month] — [intent tier: direct/faith-seeking/gift/site]
- Recommendation: [Create dedicated LP variant] OR [Use existing page]

Audience Insights:
- Search terms reveal [insight about user intent / pain point / location pattern]
- Implication for UX: [landing page messaging/design opportunity]

Geo Performance:
- [Country/region] converting best — [X% conversion]
- [Country/region] converting worst — [X% conversion] — Reason: [messaging/UX fit]
```

---

### **Friday — Joint Review & Strategy (Both Agents)**

**The 30-Minute Sync:**

1. **Traffic & Conversion Review (10 min)**
   - PS shares: "We drove 500 clicks to [page] — [X% converted"
   - UX responds: "Good, no major changes this week. Speed is solid."
   - OR: "Conversion dropped 15% — we changed hero image Tuesday. Rolling it back."

2. **Friction → Performance Loop (10 min)**
   - UX: "Mobile form abandonment is high. We're testing 3-field form instead of 6."
   - PS: "Mobile traffic is 70% of volume. This matters. Let's commit to mobile-first landing page next week."
   - Outcome: Agree on timeline for mobile LP variant; PS prepares to launch mobile-specific campaign once ready

3. **Test Results & Rollout (5 min)**
   - UX: "CTA button text 'Place My Prayer' beat 'Submit Prayer' by 12%."
   - PS: "Great. I'll update all ad copy to echo 'Place' language to set expectation."
   - Outcome: Cross-channel consistency improves

4. **Emerging Priorities (5 min)**
   - Are we working on the right landing pages? Do we need new ones?
   - Is traffic quality good? Are we reaching the right audience?
   - What's the biggest bottleneck right now — ad targeting, landing page, or conversion funnel?

**Output:**
- Agree on #1 priority for next week (e.g., "Optimize mobile form" or "Create prayer-for-healing landing page")
- Document any cross-channel changes (e.g., "Updating ad copy to match new CTA text")
- Identify any blockers or risks

## When to Loop Each Other In (Ad-Hoc)

### **UX → Paid Search (Urgent)**

**If conversion drops >10% suddenly:**
- Alert immediately (don't wait for Friday sync)
- Diagnosis: "We changed [element] on Tuesday. Rolling it back now."
- OR: "No changes from us. This is traffic quality issue — could be your targeting."

**If a landing page breaks:**
- Alert immediately
- Provide: Link to page, screenshot of issue, user impact
- Paid Search may pause keywords sending to broken page while UX fixes

**If mobile converts significantly better than desktop (>2x difference):**
- Share data with analysis
- Recommend: "Let's create mobile-specific landing page and mobile-only campaign"
- Timeline: When mobile variant is ready

**If form abandonment spikes:**
- Share which field is causing the drop
- Hypothesis: "Users can't understand what [field] is asking"
- Ask for help: "Does your ad messaging set expectations about what they'll need to enter?"

---

### **Paid Search → UX (Urgent)**

**If traffic increases but conversion stays flat:**
- Alert immediately
- Hypothesis: "We're driving more traffic. Either page isn't scaling or ad messaging doesn't match page."
- Ask: "Can you review page against these ad headlines? Is promise being fulfilled?"

**If certain keyword drives traffic but bounces at 40%+:**
- Alert immediately
- Hypothesis: "Ad messaging doesn't match page OR page is too slow OR wrong landing page for this intent"
- Ask: "Can you check load time and review page copy against [specific ad headlines]?"

**If Quality Score tanks:**
- Alert immediately
- Diagnosis: "QS issue is landing page experience. Let's schedule call to review [specific issues]"
- PS provides: Screenshot of Quality Score diagnostics

**If a landing page gets great conversion but new keywords come in:**
- "We're seeing new searches for [keyword]. This page converts well for similar intent — can we use it? Any optimizations needed?"

**If remarketing audience performs 2x better than cold traffic:**
- "Warm audiences are converting great. Let's increase budget to remarketing while we improve cold traffic landing pages."

**By the 5th of each month, both agents complete:**

- [ ] **UX:** Monthly performance summary with top wins, friction points, test results

## Data Exchange Framework

### **What UX Shares with Paid Search**

1. **CLARITY Data** (Microsoft Clarity heatmaps + session recordings)
   - Where users click, scroll, get stuck
   - Session recordings of users who dropped off
   - Heat maps showing attention patterns by page
   - Frequency: Real-time; weekly summaries in Monday brief

2. **Conversion Funnel Data** (GA4)
   - Drop-off rates by step
   - Conversion rates by device (mobile vs desktop)
   - Time on page by segment
   - Frequency: Weekly in Monday brief

3. **A/B Test Results** (Conversion data)
   - Winner with confidence level
   - Impact on conversion rate
   - Rollout plan
   - Frequency: Weekly in Monday brief

4. **Performance Metrics** (Monthly report)
   - Page load time, bounce rate, scroll depth
   - Form completion rate, field-by-field abandonment
   - Mobile vs desktop comparison
   - Frequency: Monthly end-of-month report

---

### **What Paid Search Shares with UX**

1. **Search Terms Report** (Google Ads)
   - What keywords are driving traffic
   - Which keywords have high bounce rate (page/message mismatch)
   - New emerging keywords to optimize for
   - Frequency: Weekly in Wednesday brief

2. **Traffic Quality Data** (GA4 + Google Ads)
   - Conversion rate by campaign
   - Conversion rate by keyword
   - Geo performance
   - Device performance
   - Frequency: Weekly in Wednesday brief

3. **Quality Score Issues** (Google Ads)
   - Which landing pages have low QS due to page experience
   - Expected CTR issues (ad-keyword mismatch)
   - Recommended fixes
   - Frequency: Weekly in Wednesday brief; urgent if critical

4. **Campaign Performance** (Monthly report)
   - Budget allocation by campaign
   - CPA trends
   - Impression share and competitive position
   - Seasonal insights
   - Frequency: Monthly end-of-month report

## Escalation Protocol

### **Level 1: Can Solve Independently**
- UX tests new form → rolls out winning variant → informs PS in Monday brief
- PS tests new ad copy → rolls out winner → mentions in Wednesday brief
- No cross-team approval needed; share results afterward

### **Level 2: Needs Input, Can Coordinate**
- UX wants to redesign landing page → asks PS: "Which keywords use this page?" → PS provides traffic volume and conversion rates → UX prioritizes
- PS wants to launch new campaign to emerging keyword → asks UX: "Do we have a page for [intent]?" → UX says yes/no/suggests variant
- May need 1-2 async exchanges; Friday sync covers decisions

### **Level 3: Needs Alignment Before Action**
- UX proposes mobile-only landing page redesign → needs PS to commit to mobile-specific campaign → requires agreement
- PS wants to shift budget away from low-converting page → UX needs to know why before troubleshooting
- Requires synchronous discussion (call, not email)

## Example Collaboration Scenarios

### **Scenario 1: Conversion Drops 15%**

**Timeline:** Tuesday, 3 PM
- UX notices: "Conversion on homepage dropped 15% since we launched new hero image yesterday"
- UX action: Immediately alerts PS: "Rolling back hero image. Not a traffic quality issue."
- PS response: "Good to know. I'll monitor whether conversion rebounds with rollback."
- Friday sync: "Conversion recovered. Sticking with old hero image. New one isn't resonating."
- Outcome: UX tests hero image variations methodically; PS watches for recovery

---

### **Scenario 2: New High-Converting Keyword Emerges**

**Timeline:** Wednesday morning
- PS notices: "[Keyword: "prayer for cancer healing"] driving 50+ searches/month with 15% conversion"
- PS thinks: "This intent tier is performing great. Can we scale?"
- PS asks UX: "Do we have landing page optimized for healing prayer intent?"
- UX responds: "Our prayer submission page mentions healing. Can create variant optimized for this intent if you commit budget."
- PS: "Yes — this converts. Let's create dedicated landing page and campaign."
- Timeline: UX creates variant by next Monday; PS launches campaign by Wednesday
- Outcome: Dedicated funnel for high-intent keyword; better message match; higher conversion

---

### **Scenario 3: Mobile Converts Much Better Than Desktop**

**Timeline:** Friday sync
- UX shares: "Mobile converting at 8%, desktop at 4%. Mobile form is simpler and faster."
- PS notes: "Mobile is 70% of our traffic. This is huge."
- UX proposes: "Let's make mobile-first the default. Create desktop variant if needed."
- PS proposes: "I'll shift budget to mobile campaigns. Can we launch mobile landing page by next week?"
- Agreement: UX prioritizes mobile variant; PS shifts 60% of budget to mobile campaigns
- Timeline: Mobile variant live by Thursday; new campaign structure by Friday
- Outcome: Better conversion overall; mobile is optimized; desktop gets attention next

---

### **Scenario 4: Low-Converting Page Continues Driving Traffic**

**Timeline:** Wednesday brief
- PS reports: "[Page X] getting 100 clicks/week but only 3% convert. CPA is $200."
- PS asks: "Should we pause this campaign? Is the page broken?"
- UX investigates: Reviews Clarity data, finds users scroll hero but don't click CTA
- UX insight: "Page loads fast, but CTA messaging doesn't match ad promise. Ad says 'prayer for healing' but page is generic."
- UX recommendation: "Create dedicated healing prayer landing page. This page isn't designed for that intent."
- PS action: "I'll create new campaign targeting healing keywords to new page once ready."
- Timeline: UX creates new page by Monday; PS launches by Tuesday
- Outcome: Old page gets paused; new intent-specific page drives better conversion

## Quarterly Deep Dive

Once per quarter (Jan, Apr, Jul, Oct), both agents spend 2 hours together:

1. **Review Quarter's Data** (30 min)
   - What worked? What didn't?
   - Biggest wins and biggest surprises
   - Conversion trends

2. **User Behavior Analysis** (30 min)
   - UX shares: Top friction points from Clarity data + session recordings
   - PS shares: User intent patterns from search terms
   - Identify disconnect: "Are we attracting the right people to the right pages?"

3. **Strategy Alignment** (30 min)
   - Are our priorities aligned?
   - Should we shift focus (more mobile? more healing prayer? new geo?)?
   - What should next quarter look like?

4. **Update Memory Files** (30 min)
   - Document biggest learnings in `M-memory/learning-log.md`
   - Update `M-memory/decisions.md` with strategic choices
   - Update `M-memory/feedback.md` with audience insights

## Red Flags — When to Escalate

If any of these happen, both agents stop and escalate to project lead:

1. **Conversion drops >20%** — Something structural is broken
2. **Quality Score falls below 5** — Landing page experience issue needs immediate attention
3. **Traffic spikes but conversion stays flat** — Targeting may be off-target
4. **Mobile vs desktop conversion diverges >3x** — Mobile-first strategy needed
5. **New keywords emerging faster than UX can create landing pages** — Capacity issue; need prioritization
6. **Paid Search budget running out / underspent** — ROI question; need strategy discussion
7. **Funnel has drop-off >30% at single step** — Structural design flaw
